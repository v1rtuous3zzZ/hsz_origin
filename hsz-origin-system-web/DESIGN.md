# 沪苏浙溯源平台前端设计规范

本文件是 `hsz-origin-system-web` 的唯一视觉与交互规范。所有前端 UI 新建、修改和重构都必须先阅读本文件。

本规范基于 IBM Carbon Design System 的企业风格，但已针对本项目的内网报表后台场景重新整理。不得直接套用 IBM 官网的营销页结构、Hero、CTA Banner、大字号展示区或宣传型卡片。

## 1. 产品定位

- 系统名称：沪苏浙溯源平台。
- 使用者：路公司管理人员。
- 主要内容：登录、固定菜单、查询条件、数据表格、分页、Excel 导出，后续可能增加少量 ECharts 图表和独立数据大屏。
- 运行环境：内网，不允许依赖互联网资源。
- 主要分辨率：1920×1080；兼容 1366×768 及常见桌面分辨率。
- 浏览器：Chrome、Edge、Chromium 内核的 360 极速模式。
- 不考虑移动端。

设计目标：清晰、紧凑、克制、稳定。优先保证查询效率、表格可读性和长期维护，不追求营销感或视觉装饰。

## 2. 设计原则

1. **扁平企业布局**：使用边框和背景层级区分区域，不使用卡片阴影。
2. **方角优先**：按钮、输入框、表格、侧栏和内容容器默认 `0` 圆角；系统提示最多允许 `2px`。
3. **单一品牌色**：IBM Blue 是唯一主要强调色，不创建第二套品牌色。
4. **信息密度适中**：比营销网站紧凑，比传统高密度老后台更易读。
5. **表格优先**：页面空间首先服务查询条件、表格和分页。
6. **离线可用**：禁止 CDN、在线字体、在线图标、外部图片和其他运行时网络依赖。
7. **一致性优先**：相同功能必须使用相同尺寸、间距、状态和交互。

## 3. 禁止事项

- 渐变背景。
- 玻璃拟态、毛玻璃和透明叠层。
- 大面积阴影。
- 8px 以上的大圆角。
- 胶囊按钮作为常规按钮。
- 彩色装饰卡片、营销插画、无业务意义的动画。
- 大标题 Hero、宣传型 CTA、客户 Logo 墙、营销页 Footer。
- 页面级固定宽度，例如 `width: 1920px`。
- 从外网加载字体、图片、SVG、脚本或样式。
- 在各页面散落重复颜色值和 Element Plus 深度覆盖。

  --color-surface-hover: #e8e8e8;

要求：

- 成功、警告、错误颜色只表达状态，不作为装饰。
- 表格数字和正文默认使用 `--color-text-primary`，不要按数值随机着色。

## 5. 字体和排版

字体栈：

```css
font-family:
  "IBM Plex Sans",
  "Microsoft YaHei",
  "PingFang SC",
  Arial,
  sans-serif;
```

字体要求：

- 仓库存在合法的 IBM Plex Sans 本地字体文件时才使用 `@font-face`。
- 不得从互联网动态加载字体。
- 没有本地 IBM Plex Sans 时直接使用中文系统字体回退，不阻塞开发。

后台字号：

| 用途 | 字号 | 字重 | 行高 |
| --- | ---: | ---: | ---: |
| 页面标题 | 24px | 400 | 32px |
| 区域标题 | 18px | 400 | 26px |
| 正文、表单、表格 | 14px | 400 | 22px |
| 强调文字、表头 | 14px | 600 | 22px |
| 辅助说明 | 12px | 400 | 18px |
| 数字统计 | 14px | 400/600 | 22px |

规则：

- 后台业务页面不得使用 32px 以上标题。
- 中文正文不强制使用营销规范中的大字号和 300 字重。
- 数字列使用等宽数字特性时，可设置 `font-variant-numeric: tabular-nums`。

## 6. 间距与尺寸

使用 4px 基础网格：

```css
--space-1: 4px;
--space-2: 8px;
--space-3: 12px;
--space-4: 16px;
--space-5: 24px;
--space-6: 32px;
```

布局尺寸：

```css
--header-height: 48px;
--sidebar-width: 224px;
--sidebar-collapsed-width: 56px;
--page-padding: 24px;
--page-padding-compact: 16px;
--control-height: 40px;
--table-row-height: 40px;
--border-radius: 0px;
```

- 1920×1080 默认页面边距 `24px`。
- 1366×768 或较窄窗口使用 `16px`。
- 查询条件之间使用 `16px` 水平间距、`12px` 垂直间距。
- 内容区之间一般使用 `16px` 或 `24px`，不得使用营销页的 48–96px 大间距。

## 7. 应用整体布局

### 7.1 顶部栏

- 高度 `48px`。
- 白色背景，底部 `1px` 边框。
- 左侧：菜单折叠按钮、当前页面标题。
- 右侧：用户名、退出登录。
- 不显示消息中心、主题切换、语言切换、全屏和无业务意义的工具。

### 7.2 左侧菜单

- 展开宽度 `224px`，折叠宽度 `56px`。
- 背景 `#161616`。
- 展开时显示“沪苏浙溯源平台”。
- 菜单项高度建议 `48px`。
- 普通文字为浅灰，Hover 为深灰背景，当前项使用 IBM Blue。
- 当前项不得只依赖文字颜色，需有明确背景或左侧指示条。
- 菜单固定配置，不设计动态菜单和权限占位。

### 7.3 主内容区

- 页面背景 `#f4f4f4`。
- 主业务面板背景白色，使用 `1px solid #e0e0e0`。
- 无阴影、无大圆角。
- 页面结构建议：页面标题/说明 → 查询区 → 表格区 → 分页。
- 整个页面不应产生无意义横向滚动；宽表只在表格容器内部滚动。

## 8. 登录页

- 系统标题：沪苏浙溯源平台。
- 桌面端可采用左右分栏：左侧 IBM Blue 品牌区，右侧白色登录表单。
- 没有背景图时使用纯色或简单几何背景，不引用外网图片。
- 登录表单宽度建议 `360–420px`。
- 输入框、按钮保持方角和 40–48px 高度。
- 不使用悬浮阴影卡片。
- 错误提示放在表单附近，避免弹窗打断输入。
- 1366×768 下表单必须完整可见。

## 9. 查询区

查询区是报表页面的首要操作区域。

- 使用白色背景和 `1px` 边框。
- 不使用 `el-card` 默认阴影。
- 标签与控件对齐，默认一行排列；较窄窗口允许换行。
- 控件高度统一为 `40px`。
- 主操作顺序：查询、重置、导出 Excel。
- 查询按钮使用 IBM Blue 主按钮。
- 重置使用次级或幽灵按钮。
- 导出使用次级按钮；接口未完成时应禁用并说明原因。
- 修改筛选条件不自动请求，必须由用户点击查询。

推荐排列：

```text
方向名称 | 时间区间 | 时间维度 | 查询 | 重置 | 导出Excel
```

## 10. 表格

所有常规报表使用 Element Plus `el-table`。

- 表格外边框和单元格分隔线使用 `#e0e0e0`。
- 表头背景 `#f4f4f4`，文字 `#161616`，字重 600。
- 行高建议 `40px`，表头高度 `40–44px`。
- 默认不使用彩色斑马纹；需要斑马纹时仅使用极浅灰。
- Hover 行背景使用 `#e8e8e8`，不要使用蓝色填充。
- 时间列居中或左对齐，文本列左对齐，数字列右对齐。
- 数字使用千分位，空值统一显示 `--`。
- 长文本省略并提供 Tooltip。
- 不添加无业务意义的操作列、勾选框、编辑和删除按钮。
- 空状态文案：“暂无符合条件的数据”。
- 加载状态使用表格 Loading，不用全屏遮罩。
- 错误状态保留查询条件，并提供重新查询入口。

分页：

- 放在表格下方，距离表格 `16px`。
- 显示总条数、页码、每页数量。
- 默认每页 20 条，可选 20、50、100。
- 位置保持一致，建议右对齐。

## 11. 按钮

### 主按钮

- 背景 `#0f62fe`，文字白色。
- Hover `#0050e6`，Active `#002d9c`。
- 无圆角、无阴影。

### 次级按钮

- 白色或深色背景，必须有明确边框。
- 不能与主按钮争夺视觉层级。

### 危险按钮

- 仅用于真实危险操作。
- 本期报表页面原则上不应出现危险按钮。

### 禁用按钮

- 明确降低对比度。
- 仍需满足文字可读性。
- 不可仅通过透明度使文字完全不可辨认。

## 12. 输入框、选择器和日期控件

- 高度统一 `40px`。
- 方角。
- 默认浅灰背景或白色背景配底边/边框，但同一页面必须统一。
- Focus 使用 IBM Blue 边框或 `2px` 焦点轮廓。
- Placeholder 使用 `#8d8d8d`。
- 错误状态使用 `#da1e28`，同时显示文字说明，不能只靠颜色。
- 多选标签尽量紧凑；不得使用过度圆润的胶囊风格。

## 13. 提示和状态

- 成功提示：绿色，仅用于已完成操作。
- 警告提示：黄色或浅黄背景，用于数据覆盖限制、密码待修改等状态。
- 错误提示：红色，用于请求失败和校验失败。
- 信息提示：IBM Blue，用于说明性信息。
- 页面固定说明优先使用内联提示，不频繁弹窗。
- 不可达数据不得伪造为 0，应明确显示“数据不可达”或禁用选项。

## 14. ECharts 约束

第一期可不使用图表。后续添加图表时：

- 默认白色背景、无阴影、细边框。
- 主系列优先使用 IBM Blue。
- 多系列颜色需采用低饱和、可区分配色，不使用彩虹色。
- 网格线使用浅灰。
- 图表标题不超过 18px。
- 图表必须有与表格一致的数据口径说明。
- 不使用 3D、发光、粒子和无业务意义的动画。

## 15. 响应式与缩放

- 最小目标宽度为 1366px，不设计移动端断点。
- 1366px 下查询区允许换行，侧栏可折叠。
- 主内容宽度使用弹性布局，不能写死。
- 表格列较多时使用内部横向滚动和固定关键列。
- 页面高度不足时主内容区域可纵向滚动，顶部栏保持稳定。
- 浏览器缩放 90%–125% 时不能出现控件重叠、文字截断或分页不可见。

## 16. Element Plus 实施规则

- 保留 Element Plus 作为组件库，不另装 Carbon Vue 组件库。
- 通过全局 SCSS Token 和集中主题覆盖实现 Carbon 风格。
- 不在每个页面重复覆盖按钮、输入框、表格基础样式。
- 允许针对具体业务组件做局部样式，但不得创建第二套颜色和间距体系。
- `el-card` 如使用，必须关闭阴影并改为方角细边框；普通报表容器优先使用语义化 `section`。

## 17. 开发验收清单

每次前端 UI 变更至少检查：

- 是否使用了本文件的颜色和尺寸令牌。
- 是否存在阴影、渐变、大圆角或营销页元素。
- 1920×1080 是否清晰紧凑。
- 1366×768 是否无严重错位。
- 查询区是否可换行。
- 表格是否只在内部横向滚动。
- 表头、数字和空值是否统一。
- Focus、错误和禁用状态是否清楚。
- 是否产生任何外网资源请求。
- `npm run build` 是否通过。

若业务需求与本文件冲突，先更新本文件并说明原因，再修改 UI。
